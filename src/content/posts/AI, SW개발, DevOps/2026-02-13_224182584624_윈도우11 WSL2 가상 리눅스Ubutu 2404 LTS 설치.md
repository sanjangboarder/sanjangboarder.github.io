안녕하세요 산장보더입니다.

​

이번 포스팅은 제가 AI에 관심이 많은데 OpenClaw 라고 들어 보셨는지 모르겠는데 PC나 서버 자원에 제어권을 주는 AI Agent가 한번 써보고 싶어서 윈도우 환경에서 VM(Virtaul Machine) 으로 리눅스 환경을 설치해본 후기입니다.

​

오픈클로 몰트봇이란 이름으로 2월초부터 폭팔적으로 관심을 받고 있는데요.. 쉽게 이야기 하면 자비스의 초기형태 수준의 자율화된 AI Agent 오픈소스 라고 보심 됩니다. 이게 돌아가는 OS 환경이 현재는 Mac 이나 Linux에 적합하게 되어 있어서 잠시 Apple Mac mini M4를 충동구매 할까도 생각했지만 그냥 가볍게 VM으로 리눅스를 깔아볼려고 마음을 접었습니다.

​

대부분 윈도우 환경에서 PC 사용을 하실텐데요. 윈도우에서 리눅스를 올리기 위해서는 VM 이나 Container 형태로 만드는 방법이 있는데 제가 알던 쉬운 방법은 VMware 정도를 쓰는건데 찾아보니 윈도우에 WSL2 라고 해서 리눅스를 가상화된 환경으로 쉽게 올리는게 있떠라구요.. 그래서 바로 한번 실행해 봤습니다.

---

설치는 무척 간단한데요. 윈도우 PowerShell 콘솔을 열고 wsl --install 만 쳐주면 바로 설치가 됩니다. 이후에 리부팅 한번 해주면 되구요.

[![](https://mblogthumb-phinf.pstatic.net/MjAyNjAyMDVfMjkx/MDAxNzcwMjk4MjAxNzQ0.m-apvYGMaLDQ9O-jOzB_GoLSIXOiCgQubiEcbqC4oJ0g.97ydy_ktQeDIkXKaHnj2k3qRjNc04hYi9xVPKsQsAE0g.PNG/image.png?type=w800)](#)

WSL이 설치되면 MicroSoft Store에서 Ubuntu 를 검색하면 몇가지가 나오는데 저는 최근 안정된 최신 버전은 24.04 LTS를 골랐습니다. 그냥 스토어에서 검색해서 설치만 하면 놀랍게도 바로 설치가 되더군요.. 세상 참 좋아졌네요.

[![](https://mblogthumb-phinf.pstatic.net/MjAyNjAyMDVfMTI0/MDAxNzcwMjk4ODYyMDI0.alFKUMkbf3SH3kMbgyguM4Qf2pjoeioFzZq8fypsMg4g.Yjn4GdSyuFVi6OLv_xvX9rADZRnafLSj84_cab70pxAg.PNG/image.png?type=w800)](#)

우분투를 스토어에서 설치하면 WSL 관련된 설정을 몇가지만 해주면 되는데요.. RAM이나 CPU, Storage 할당 등을 해줄 수 있습니다. 잘 모르는 내용이면 Default Setting을 이용해도 큰 문제는 발생하지 않을겁니다.

[![](https://mblogthumb-phinf.pstatic.net/MjAyNjAyMDVfMjMw/MDAxNzcwMjk4ODY5NTY0.dGLXbpAb-zmjdN72uVmTUMWpthyoGxzgysKxr9TxYUkg.U9koe0lGFMQP_0hOB0ewUi9W67kdM3EFIejzBCoiWmwg.PNG/image.png?type=w800)](#)

MS의 자세한 설명은 아래 링크에서 찾아보 실 수 있습니다.

관련 문서 : <https://learn.microsoft.com/ko-kr/windows/wsl/>

혹시 Core 수나 이런걸 조정할 수는 있는데 저는 i5-12600K cpu를 쓰는데 물리적인 core는 10개이고 Thread로 보면 16개로 되어 있어서 WSL에서 프로세스 수 설정은 16개 까지 가능하도록 되어있습니다. 우분투를 덩치 큰 서비스를 돌릴게 아니기 때문에 4core면 일반적인 용도에서는 충분할것이고 부족하다고 생각하시면 늘리시면 됩니다.

​

메모리 크기는 보통 core 수 x 2 정도가 적당하니 4 core 할당하면 8G 정도 주면 되구요.. 스토리지 크기는 스왑크기로 지정하면 되는데 가상파일시스템이 위치하는 내 로컬 드라이브의 여유공간 크기보다 작게 설정하면 됩니다.

​

스토리지는 굳이 설정하지 않았다면 1T정도로 잡히는데 내 실제 물리적인 여유공간이 200G인데도 1T로 잡더라구요.. 이건 가상으로 잡혀있는 공간이라 실제 물리적인 공간에 데이터가 들어가면 더 이상 WSL 우분투에서도 사용이 불가능할거라 이왕이면 맞춰주는게 좋습니다.

[![](https://mblogthumb-phinf.pstatic.net/MjAyNjAyMDVfODkg/MDAxNzcwMjk5MTI3Nzcz.vAj6rwEX2cit3X8RKwNhAjektS5Wgr4xubGFmljZ5acg.oHM1RW6tQIKq5oDG71XIuZpVdibBj6J1FF-e8jC6n00g.PNG/image.png?type=w800)](#)

WSL2 로 쉽게 우분투를 설치했는데.. IDE 환경으로 설치가 된건 아니더군요.. 그래서 콘솔을 띄워서 사용하도록 되어 있는데요.. 제가 나름 SW개발자 출신이라 저는 익숙한데 이게 불편하실 분들도 있긴 할겁니다. 하지만 GUI 환경으로 사용도 가능한 방법이 있기 때문에 용도에 따라 쓰심 될듯 싶어요..
